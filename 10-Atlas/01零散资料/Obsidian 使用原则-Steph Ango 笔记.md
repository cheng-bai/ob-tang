# Obsidian 使用原则-Steph Ango 笔记

> 来源：Steph Ango《How I organize my Obsidian vault》
> https://stephango.com/vault

---

## 核心理念

### 1. 文件优于应用 (File over App)
- 使用你能掌控的格式（Markdown 文件）
- 易于检索和阅读，不依赖特定软件

### 2. 拥抱混乱与懒惰 (Embrace Chaos and Laziness)
- **自下而上涌现结构**，而非自上而下的 rigid 规划
- 不要预先过度分类
- 让组织方式自然生长

### 3. 一致性减少决策 (Consistency Collapses Decisions)
- 建立个人规则，避免反复决策格式和组织问题
- 形成习惯后，不再纠结"这个笔记该放哪里"

---

## 组织结构

### 文件夹使用原则

**核心：少用文件夹**

| 推荐结构 | 用途 |
|---------|------|
| **根目录** | 个人笔记（日记、随笔、常青笔记） |
| references/ | 外部内容（书籍、电影、地点、人物） |
| clippings/ | 他人写的内容 |
| attachments/ | 媒体文件 |
| daily/ | 日记（YYYY-MM-DD.md 格式，也可放根目录） |
| templates/ | 模板文件 |

**关键做法：**
- 不用嵌套子文件夹
- 不用文件夹做分类组织
- 大部分笔记都在根目录
- 用快速切换器 (Cmd/Ctrl+O)、反向链接、内部链接导航
- 不用文件浏览器找笔记

---

## 内部链接

### 链接原则
- **大量使用内部链接** — 首次提到某事物时就加链接
- **未解析的链接也有价值** — 它们是未来连接的线索
- 重度链接帮助追溯想法如何随时间涌现

### 示例（数学教学场景）
[[函数]] 的 [[单调性]] 可以通过 [[导数]] 来判断...

## 个人规则建议

| 规则 | 具体做法 |
|------|---------|
| 单一库 | 保持在一个 vault 中，不要拆分 |
| Markdown | 避免非标准 Markdown 语法 |
| 标签复数 | #lessons 而非 #lesson |
| 日期格式 | 统一使用 YYYY-MM-DD |
| 评分系统 | 7 分制（高端更精细） |
| 待办清单 | 每周一个总清单，而非每日分散 |

---

## 笔记与模板

### 属性设计原则
- **跨类别复用**：如 genre 可用于书籍、电影、剧集
- **可组合**：一个笔记可用多个模板
- **短属性名**：start 而非 start-date（为了输入速度）
- **列表类型优先**：更灵活

## 维护实践

### 分形日记 (Fractal Journaling)
全天 -> 写片段 -> 每隔几天回顾 -> 每月汇总编译 -> 每年年度回顾

### 随机回顾
- 定期使用随机笔记功能
- 重新发现旧想法
- 补充缺失的链接

### 手动维护
- 不要过度自动化理解过程
- 手动整理帮助理解自己的模式

## 对我的建议（数学教师场景）

### 当前问题
- 使用编号文件夹 (00-, 01-, 02...) 过度组织
- 依赖文件夹导航而非链接
- 可能预先过度分类

### 改进方向
1. **扁平化结构** — 将教学笔记移到根目录
2. **创建模板** — 备课/概念/反思模板
3. **增加链接** — 数学概念之间建立网络
4. **信任搜索** — 用 Cmd/Ctrl+O 快速切换
5. **保持日记习惯** — 定期汇总教学反思
