# AI-CONTEXT

> 本文件为 AI 编码代理/助手提供关于此 Obsidian 知识库的上下文和指令
> 格式参考：https://agents.md/

---

## 知识库概述

这是一个**高中数学教师**的个人知识管理系统，使用 Obsidian 构建。

**核心理念**（参考 Steph Ango）：
- 文件优于应用 — 使用标准 Markdown
- 拥抱混乱 — 自下而上涌现结构，不过度分类
- 一致性减少决策 — 建立个人规则

## 笔记风格约定

### YAML Frontmatter
```yaml
---
type: [lesson|concept|reflection|daily|template]
tags: [数学，教学，...]
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

### 数学公式
- 行内公式：`$E = mc^2$`
- 块级公式：`$$\int_a^b f(x)dx$$`

### 内部链接
- 使用 `[[笔记名]]` 格式
- 首次提到某概念时就加链接
- 示例：`[[函数]] 的 [[单调性]]`

### 标签规范
- 使用复数形式：`#lessons` 而非 `#lesson`
- 层级标签：`#数学/代数`、`#数学/几何`

## AI 助手工作指南

### 当用户请求帮助时

1. **创建教学材料**
   - 先确认年级、课题、课时
   - 参考现有教案风格
   - 添加适当的数学公式

2. **整理笔记**
   - 保持扁平结构，避免过度嵌套
   - 添加内部链接连接相关概念
   - 使用 YAML 属性标记类型和标签

3. **搜索内容**
   - 优先使用内部链接追溯
   - 利用标签过滤
   - 随机回顾发现旧想法

### 输出格式偏好
- 使用清晰的 Markdown 结构
- 数学内容必须用 `$` 或 `$$` 包裹
- 列表项保持简洁
- 代码块标注语言类型

## 维护实践

### 定期任务
- 每周：汇总教学反思
- 每月：编译月度回顾
- 每年：年度总结

### 链接维护
- 新笔记创建时添加相关链接
- 定期使用"未解析链接"功能补充连接
- 随机回顾旧笔记，建立新连接
