# AGENTS

> 本文件供 AI 代理参考，帮助理解承白老师的 Obsidian 教学知识管理系统

---

## 项目概述

这是一个**高中数学教学知识管理系统**，用于：
- 管理教学资源（试卷、讲义、题目）
- 构建知识体系（高中数学知识点）
- 自动化输出（Markdown 转 PDF 打印）
- AI 辅助教学

## 工作日志记录规范

### 位置
`30-Log/Insights/AI-工作日志.md`

### 记录格式
以后所有的 AI 操作日志**不再单独创建碎片文件**，而是**统一追加（Append）写入到同一个文件**中，按日期倒序排列（最新的在最上面）。

### 内容要求
每一次协助优化需用 Markdown 分级标题注明：
- `## YYYY-MM-DD` 时间戳
- 记录 AI 执行的具体操作与优化
- 列出主要变更的文件
- 说明待办事项

## 打印输出规范

### 工具
`Skills/markdown-print-kit/`

### 题块语法
```markdown
:::space type=mc level=A lines=2
:::

:::space type=fill level=B lines=3
:::

:::space type=solve level=C lines=10
:::
```

### 参数说明
- `type`: `mc`（选择）| `fill`（填空）| `solve`（解答）
- `level`: `A`（基础）| `B`（中档）| `C`（难题）
- `lines`: 留白行数

## AI 交互原则 (安全红线)

1. **先确认理解，再执行**
2. **重要操作前询问确认**：尤其是批量修改或删除内容时。
3. **安全操作区**：严禁直接修改 `.obsidian/` 隐藏配置文件夹的内容（除非用户明确要求）。禁止主动删除 `10-Atlas/` 中的 PDF 原生资料文件。
4. **复杂任务先列计划**
5. **严谨至上**：数学公式和知识内容必须准确无误。
6. **主动进行版本控制**：当完成系统重构、批量修改目录、或生成大量重要讲义内容后，AI 应该主动询问用户是否需要执行 Git commit 与 push 同步。

## 常用操作示例

### 创建新讲义
1. 复制 `00-System/Templates/讲义/新授课讲义模板.md`
2. 填写模板变量
3. 保存到 `20-Product/01-数学教学/讲义/`

### 添加题目索引
1. 使用 `00-System/Templates/通用/题目收集模板.md`
2. 填写 frontmatter（tags、难度、知识点）
3. 保存到 `20-Product/01-数学教学/1-题目集/`

### 查询知识资源 (全局导航锚点)
1. **全局寻路入口**：打开仓库根目录的 `🏠-首页.md`
2. **知识点与试题检索**：打开 `10-Atlas/05-知识汇总-数学知识按主题分类/📊-知识索引中心.md`
3. 使用 Dataview 进行代码检索。
