# 🏷️-标签体系说明

> 本文档定义了本知识库的标准标签体系，请严格遵守以确保 Dataview 查询正常工作。

---

## 📋 标签层级结构

### 1️⃣ 类型标签 (必打)

用于区分文件的基本类型：

```
#类型/题目        # 单个题目
#类型/讲义        # 讲义资料
#类型/试卷        # 完整试卷
#类型/知识点      # 知识点卡片
#类型/笔记        # 一般笔记
#类型/MOC         # 地图类文件
#类型/说明文档    # 说明文档
#类型/模板        # 模板文件
```

## 🔍 Dataview 查询示例

### 查询所有函数相关题目

```dataview
TABLE 
  difficulty as "难度",
  source as "来源"
FROM #类型/题目 AND #数学/函数
SORT difficulty DESC
```

### 查询所有高三讲义

```dataview
TABLE 
  subject as "专题",
  date as "日期"
FROM #类型/讲义 AND #年级/高三
SORT date DESC
```

### 查询所有高考真题

```dataview
TABLE 
  year as "年份",
  region as "地区"
FROM #类型/试卷 AND #来源/高考真题
SORT year DESC
```

### 查询待整理的文件

```dataview
LIST
FROM #状态/待整理
SORT file.mtime DESC
```

## ⚠️ 使用规范

1. **必打标签**: 所有文件必须至少有一个 `#类型/xxx` 标签
2. **层级清晰**: 使用 `/` 分隔层级，如 `#数学/函数` 而非 `#函数`
3. **避免重复**: 不要同时使用 `#难度/难题` 和 `#难度/压轴`
4. **统一命名**: 严格使用本文档定义的标签名称
5. **标签数量**: 建议每个文件 2-5 个标签，不宜过多

---
